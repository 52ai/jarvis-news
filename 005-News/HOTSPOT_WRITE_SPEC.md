# 舆情报告写作规范要点（v11 · 2026-05-25 固化）

> 来源：4月30日日报格式 + WEIXIN_ARTICLE_SPEC.md v27 + TOP50数据库

## 报告结构（9段，必须按顺序）

```
① AIGC frontmatter（完整保留）
② 标题：# 🦞 龙虾日记 | YYYY年M月D日 AI科技舆情热点日报
③ 副标题：（副标题：xxxx）
④ 📊 历史舆情统计（收录记录数/实体数/媒体数/今日新增/持续≥3天）
⑤ 📋 今日热点摘要（150-200字，含主线概述+关键词）
⑥ 🔥 TOP 5 舆情焦点（每条含星级/来源/类型/摘要/核心要点5条/Jarvis点评≥50字）
⑦ 🦞 Jarvis 辣评（3-5个独立观点段落，每段≥50字）
⑧ 📊 附录（今日TOP20舆情列表 + 今日TOP20媒体列表）
⑨ EOF（元数据：日期/时间/地点/出品人/诞生天数/站点/来源）
```

## TOP 5 每条结构（完整字段）

```
星级（⭐×1-5）

🔴【新】或 🟡【续】标记

摘要（≤50字）

Jarvis点评（独立技术判断，≥50字）
```

## 四维评分标准

| 维度 | 说明 |
|------|------|
| 突发性 | 是否近两日爆发 |
| 数据冲击力 | 数字具体程度 |
| 反常识程度 | 是否打破预期 |
| 叙事张力 | 故事性/人物性/戏剧性 |

---

## 禁用语清单

| 禁用语 | 正确替换 |
|--------|---------|
| 本质 | 实质 / 核心逻辑 |
| 实际上 | 实际观察 |
| 本质上 | 从技术角度看 |
| 首先其次最后 | 用时间顺序或并列结构 |
| 综上所述 | 直接给出结论 |

---

## URL 格式规范

- 位置：正文末尾，双换行前
- 格式：`URL: https://xxx.xxx`
- 禁止：`→ https://` / 内嵌同行 / 无前缀

---

## 禁止清单

- ❌ AIGC frontmatter（报告头部保留，不写入正文）
- ❌ 封面图
- ❌ 焦虑悲观叙事（AI for Good 视角）
- ❌ 「被AI取代」「饭碗被抢」核心叙事
- ❌ 无法核实的小道消息

---

## 质量三问（快速自检）

1. **阅读量**：标题有数字/反常识判断？前三行给"跟读者有什么关系"？
2. **认知提升**：有"原来是这样"？数字有翻译？
3. **十年后**：回看还值得存在吗？
