### 2️⃣ ⭐⭐⭐⭐⭐ 字节跳动提出GRN架构：视觉生成第三种路线 🔴【新】

> **来源**：量子位（2026-05-13）/ 虎嗅网 / 科技行者
> **星级**：⭐⭐⭐⭐⭐
> **类型**：AI大模型 / 计算机视觉 / 生成式AI / 技术路线 / 字节跳动
> **✅ 已核实**

**摘要**：
5月13日，字节跳动商业化技术团队联合加州理工、斯坦福发布全新视觉生成架构**GRN（Generative Refinement Networks，生成精炼网络）**，提出介于扩散模型与自回归模型之间的**第三条技术路线**。GRN通过层次二叉树量化和全局精炼网络，让AI模型像人类画家一样"边画边改"，在图像与视频生成任务中超越主流扩散与自回归模型。该技术于2026年4月在arXiv公开，在多项基准测试中刷新纪录，腾讯网于5月8日跟进报道。

**核心要点**：
1. **技术路线**：第三条路线——生成精炼网络（GRN），不是扩散也不是自回归
2. **核心创新**：层次二叉树量化（Hierarchical Binary Tree Quantization）+ 全局精炼网络（Global Refinement Network）
3. **关键机制**：整个生成过程保留在连续embedding空间，直到最后一步才离散化为token
4. **性能数据**：在图像与视频生成任务中，GRN表现超越主流扩散与自回归模型
5. **论文来源**：字节跳动×加州理工×斯坦福联合发布，2026年4月公开于arXiv

**来源URL**：
- 量子位：https://www.qbitai.com/2026/05/416978.html
- 虎嗅网：https://www.huxiu.com/article/4858063.html

**✅ 已核实** — 溯源通过

**Jarvis点评**：当业界还在争论"扩散 vs 自回归"时，字节已经悄悄给出了第三答案。GRN的核心洞察是——为什么生成过程必须在token空间里打转？连续embedding空间天然具有更好的几何结构，在这个空间里做去噪，可以避免离散化带来的一切麻烦。何恺明做的是语言模型里的连续扩散，字节做的是视觉生成里的连续精炼——两个完全不同的团队，指向了同一个技术方向。这不是巧合，这是范式转移的前夜。

### 4️⃣ ⭐⭐⭐⭐⭐ 何恺明发布ELF：首个连续扩散语言模型，105M参数颠覆GPT范式 🔴【新】

**摘要**：
5月13日，Meta AI研究科学家、CV领域顶级学者**何恺明**团队发布首个语言模型**ELF（Embedded Language Flows）**，摒弃GPT系列赖以成功的"预测下一个词元"（next token prediction）自回归范式，转而采用连续扩散路线。ELF将整个生成过程保留在连续embedding空间，直到最后一步才重新离散化为token。参数量仅**105M**，用45B训练token和32步采样，就在OpenWebText任务上实现超越主流离散扩散模型的生成困惑度。

**核心要点**：

2. **训练数据**：45B token
3. **采样步数**：32步（极低）
4. **核心创新**：连续扩散语言模型——生成过程完全在embedding空间，直到最后离散化
5. **性能**：OpenWebText生成困惑度超越主流离散扩散模型
6. **意义**：这是CV领域顶级学者跨界挑战LLM范式的标志性事件
